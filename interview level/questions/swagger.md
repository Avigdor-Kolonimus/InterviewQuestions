# Swagger

## 1. What is Swagger used for?

**Swagger** is a set of tools based on the **OpenAPI Specification (OAS)** used to design, document, test, and generate REST APIs.

### Main use cases

- Generate interactive API documentation.
- Describe API endpoints, request/response formats, and authentication.
- Test API endpoints directly from the browser using **Swagger UI**.
- Generate client SDKs and server stubs in multiple programming languages.
- Keep API documentation synchronized with the implementation.

### Typical workflow

1. Define the API using the OpenAPI Specification (YAML or JSON), or generate it from code annotations.
2. Visualize and interact with the API using **Swagger UI**.
3. Optionally generate client libraries or server code using **Swagger Codegen** or **OpenAPI Generator**.

### Benefits

- Self-documenting APIs.
- Easier collaboration between frontend and backend teams.
- Simplifies API testing.
- Reduces manual documentation effort.
- Helps ensure consistency across services.
