# Monitoring (Prometheus, Grafana)

## 1. How does Prometheus work?

**Prometheus** is an open-source monitoring and alerting system designed for collecting and storing metrics in a time-series format.

### Core idea

Prometheus works by **pulling metrics** from configured targets at regular intervals (scraping), rather than receiving pushed data.

---

## Key components

### 1. Prometheus Server
- Collects and stores metrics.
- Periodically scrapes HTTP endpoints (usually `/metrics`).

### 2. Exporters
- Applications or services that expose metrics in Prometheus format.
- Examples:
  - Node Exporter (system metrics)
  - PostgreSQL Exporter
  - Redis Exporter

### 3. Time-series database
- Stores metrics as:
  ```
  metric_name{label="value"} -> value + timestamp
  ```

### 4. PromQL
- Query language for retrieving and aggregating metrics.
- Example:
  ```promql
  rate(http_requests_total[5m])
  ```

### 5. Alertmanager
- Handles alerts generated by Prometheus.
- Sends notifications (email, Slack, etc.).

---

## How it works (flow)

1. Application exposes metrics at `/metrics`
2. Prometheus scrapes metrics periodically
3. Metrics are stored in time-series DB
4. PromQL queries analyze data
5. Alerts are sent via Alertmanager if thresholds are exceeded

---

## Key features

- Pull-based architecture
- Multi-dimensional data (labels)
- Powerful query language (PromQL)
- Built-in alerting system

---

## 2. What does Grafana do?

**Grafana** is a visualization and analytics platform used to display metrics collected by systems like Prometheus.

---

## Key functions

### 1. Visualization
- Creates dashboards with graphs, charts, and tables.
- Visualizes time-series data.

### 2. Data sources
- Supports Prometheus, InfluxDB, Elasticsearch, Loki, etc.

### 3. Dashboards
- Custom dashboards for system monitoring.
- Real-time updates.

### 4. Alerting
- Can trigger alerts based on metric conditions (often integrated with Prometheus).

---

## Prometheus vs Grafana

| Tool | Purpose |
|------|--------|
| Prometheus | Collects and stores metrics |
| Grafana | Visualizes metrics |

---

## Typical architecture

```
Application → Prometheus → Grafana
                     ↓
              Alertmanager
```

---

## Summary

- **Prometheus** = collects and stores metrics using a pull-based model + PromQL.
- **Grafana** = visualizes metrics and builds dashboards from Prometheus (and other sources).
